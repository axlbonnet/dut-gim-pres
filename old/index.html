Code C calcul de score


#include <stdio.h>

int main(void) {
        int scoreNom, scoreAnniversaire;
        int scoreFinal;

        scoreNom = 4 + 6;
        scoreAnniversaire = 15 + 4 + 1997;
        scoreFinal = scoreNom * scoreAnniversaire;
        scoreFinal = scoreFinal % 37;
        printf("Ton score : %d \n", scoreFinal);
        return 0;
}

Les données

Représentation des données

  • la RAM est constitué de bits (binary digit) (des 10aines de millards)
  • Un bit contient 0 ou 1
  • 2 bits permettent de stocker 4 valeurs différentes 00 01 10 11
  • 3 bits -> 8 valeurs
  • 4 ? 5 ? 6 ? 7 ?
  • 8 bits ? -> 256 valeurs

Représentation des données : l'octet

  • Sur 8 bits, peut prendre 256 valeurs différentes
  • Plus petit élément de stockage
  • Comment représenter l'information ?
  • On stocke les nombre en base 2
  • Rappel base 10 : 2016 = 2*1000 + 0*100 + 1*10 + 6*1
  • 75 = 1*64 + 0*32 + 0*16 + 1*8 + 1*4 + 0*2 + 1*1
  • 75 = 1001101 = 01001101
  • 2016 = 1024 + 512 + 256 + 128 + 64 + 32
  • 2016 = 11111100000

Représentation des données : en C

  • Pour dépasser 255, plusieurs octets sont nécessaires
  • Les int sont stockés sur 4 octets (dans 99,9% des cas)
  • 2016 = 00000000 00000000 00000111 11100000
  • 4 octets -> 232 valeurs = 4 294 967 296
  • les valeurs négatives ?

Les types décimaux en C

nomtailleValeurs
int4-2 147 483 648 à 2 147 483 647
unsigned int40 à 4 294 967 295

unsigned short20 à 65 535
long long8−/+ 9223372036854775807
  • Taille en octets
  • Pas exact dans 100% des cas
  • le type int nous suffira tout le temps

Représentation des données : les caractères

  • Un caractère est codé sur 1 octet (256 valeurs)
  • ASCII : code qui attribue à chaque valeur un caractère
  • Exemple : 109 = 1101101 -> 'm'
  • Très limité, pas d'accents
  • En c, le type char est en fait un entier sur 1 octet
  • char c = 'm'
  • Chaine de caractère : suite de chars
  • "coucou" <-> '99'-'111'-'117'-'99'-'111'-'117'

Représentation des données : les réels

  • Plus complexe, on va décrire une méthode qui s'en rapproche
  • En C un float sur 4 octets permet de stocker de
  • 0,0000000000000000000000000000000000000175494351
  • à 340282347000000000000000000000000000000
  • Décomposition en notation scientifique
  • 1.175494351 * 10-38 et 3.40282347 * 1038
  • signe sur 1 bit, mantisse sur 23 bits, exposant sur 8 bits

Les réels en C

  • float sur 4 octet : 10-38 et 1038
  • double sur 8 octets : 10-308 et 10308
  • long double sur 10 octets : 10-4932 et 104932
  • On utilisera que des float
  • Pas d'égalité avec des float !

Conclusion

  • Les données sont stockées dans la RAM dans des octets de 8 bits pouvant représenter 256 valeurs
  • Plusieurs formats (types) pour coder l'information
  • Il faut déclarer les variables dans un type précis
  • on utilisera uniquement des int et des float (peut-être des char)

L'opérateur modulo :

"a % b" : Reste de la division euclidienne

9 % 4 = 1

C'est l'équivalent de ce qu'il y a après la virgule dans un division "exacte"

9÷4=2.25. Le 0.25 correspond au modulo

Si le modulo vaut 0, cela veut dire que la b divise a

L'opérateur division :

"a / b" : Division euclidienne uniquement si a et b sont des int

"a / b" : Division exacte si a et/ou b est un float

Exemple : 9/4 = 2

Exemple : 9.0/4 = 2.25

Si a et b sont entier : a/b est le résultat de la division euclidienne, a%b est le reste

a = (a/b)*b + (a%b)

Applications du modulo :

"a % b" : Savoir si b divise a

Savoir à combien de minutes correspondent 442 secondes :

442 / 60 = 7

442 % 60 = 22

442 = 7*60 + 22

On veut saisir 10 entiers, puis les réafficher

#include <stdio.h>
int main(void) {
  int tableau[10];
  int i;

  for (i=0 ; i<10 ; i++) {
    scanf_s("%d", &tableau[i]);
  }
  for (i=0 ; i<10 ; i++) {
    printf("%d", tableau[i]);
  }
  return 0;
}
